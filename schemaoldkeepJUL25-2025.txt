Collection: user
Stores all user account information including contact details, permissions, assigned processes, and organizational relationships.

Fields:

_id: (String/UUID) Unique identifier for the user. Example: "d61d7027-b51e-4ed7-ba2d-24041bc6a07a"

name: (String) User's display name. Example: "newmwber02"

phoneNumber: (String) User's phone number in E.164 format. Example: "+919010054767"

email: (String/null) User's email address (optional). Example: null

Nested Objects:
5. designation: Contains job title info

_id: (ObjectId) Designation ID

name: (String) Title name. Example: "testmenber"

role: Defines user permissions

_id: (ObjectId/null) Role ID

name: (String) Role name. Example: "user"

ctmP: Custom permissions object

webAllocation: (Boolean) Web access flag

callRecording: Sub-object

play: (Boolean) Can play recordings

download: (Boolean) Can download recordings

reports: Sub-object

downloadReports: (Boolean) Report download permission

process: (Array) List of assigned workflows

Each item contains:

_id: (ObjectId) Process ID

name: (String) Process name. Example: "Default process"

type: (String) Process type. Example: "both"

company: Organization details

_id: (ObjectId) Company ID

name: (String) Company name. Example: "runo"

reportsTo: (Array) List of supervisors

Each item contains:

_id: (UUID) Supervisor's user ID

name: (String) Supervisor's name. Example: "azhartest6"

license: Software license info

_id: (ObjectId) License ID

expiryDate: (Unix timestamp) License expiration

createdAt: (Unix timestamp) License issue date

updatedAt: (Unix timestamp) Last update

Timestamps:
12. createdAt: (Unix timestamp) Account creation time
13. updatedAt: (Unix timestamp) Last profile update

Key Relationships:

Linked to company collection via company._id

Linked to process collection via process[]._id

Linked to other users via reportsTo[]._id

Common Query Patterns:

Find users by role name

Filter users with specific process assignments

Check license expiration status

Find team members under a supervisor

Special Notes:

phoneNumber always includes country code

Empty arrays mean no relationships (e.g., empty reportsTo = no supervisor)

Timestamps are in Unix format (seconds since epoch)

report-agent

collection: report-agent-login  
This collection tracks agent login sessions, call activity, and time distribution for performance reporting.  

_id: Unique identifier for the login report.  

companyId: The ID of the company this report belongs to, referencing the company collection.  

user: A dictionary containing details of the agent/user, including their ID and name.  

dateRange: A dictionary with 'start' and 'end' Unix timestamps defining the reporting period.

breakTime: Total time (in seconds) the agent spent on breaks.  

createdAt: Unix timestamp of when the report was generated.  

updatedAt: Unix timestamp of the last update to the report.  


collection: report-history  
This collection stores historical records of generated reports, including their parameters and status.  

_id: Unique identifier for the report history entry.  

cId: The ID of the company associated with the report, referencing the company collection.  

userId: The ID of the user who requested or generated the report.  

pId: The ID of the process (if applicable) related to the report, referencing the process collection.  

fromEpoch: The starting Unix timestamp (integer or float) for the report's data range.  

toEpoch: The ending Unix timestamp (integer or float) for the report's data range.  

type: A string indicating the type of report (e.g., 'agent-performance', 'call-summary', 'customer-activity').  

ids: A string or comma-separated list of IDs relevant to the report (e.g., user IDs, customer IDs).  

query: A dictionary containing the filters or query parameters used to generate the report.  

status: The current status of the report (e.g., 'pending', 'completed', 'failed').  

updatedAt: Unix timestamp of the last update to the report history entry.  

createdAt: Unix timestamp of when the report history entry was created.
